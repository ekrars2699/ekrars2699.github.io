Movie Webpage Project

Context

The Movie Webpage Project is a project where I made a site with a partner. The site we made was about a fictional movie that we came up with.

Content

The project requires us to have both HTML and CSS in it. Before we started the actual project we had to make a plan for the project. In the plan we had split up the requirements of the project. The requirements are:

HTML Requirements

  • Headings
  • Paragraphs
  • Divs and Spans
  • Google Fonts
  • Inputs and Sumbit

    • CSS Requirements

  • Selectors: element and class and id
  • Padding, Border, and margin

    • We also need at least 2 images that must be local.

    Reflection

    There were a few challenges while working on this project. We had a problem with deciding on how the site should look like. The design of our site had changed multiple times while making it. We had changed the color and the fonts used multiple times. I was in charge of the images that went on the site and at first I couldn't decide on which images I should choose. Many of the images I didn't fit the theme of the movie. After searching I was eventually able to find 2 images that related to the story of our movie. Other problems we had were related to meeting the requirements of the project, such as some of our code not working which caused us to review our notes or ask for help.

    Takeaways

    While working on this project I learned the value of having a plan. This project was the first time a plan was a requirement and it greatly helped us. The plan made it so that we could meet the requirements and so we don't stay stuck on one thing. If we didn't have a plan we wouldn't have been able to add some of the things that made our site look good, for example the background which changes color as you go down.

    Next Steps

    In the future I would like to add more details that would make our site more unique. I would also like to do the extra cedit which is to make a sequel to our movie.
